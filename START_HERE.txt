â“ QUICK Q&A

Q: Do I need an API key to start?
A: No! Free search engines (DuckDuckGo, Indeed) work without API keys.
   Paid APIs (SerpAPI, Google CSE) are optional for better results.

Q: How many results will I get?
A: Typically 50-300 jobs per search, depending on job market and keywords.

Q: How many emails will I extract?
A: Usually 5-20% of jobs have extractable emails. More jobs = more emails.

Q: Can I use this for recruitment?
A: Yes! Perfect for finding candidate emails and reaching out directly.

Q: Is this free?
A: Yes! Completely free with free search engines. Optional paid APIs available.

Q: How many emails can I send per day?
A: Maximum 50 emails per day (enforced to prevent spam).

Q: Where are results saved?
A: In the output/ folder:
   - output/web_jobs_ultimate.json (all jobs)
   - output/web_jobs_ultimate.txt (readable summary)
   - output/found_emails.csv (extracted emails)
   - output/scrape_archive.json (all history)

Q: What if I don't get results?
A: 1. Increase "Max results per query" slider
   2. Try different/broader keywords
   3. Check internet connection
   4. Check detailed guide: GUI_USER_GUIDE.md

Q: Can I run it without the GUI?
A: Yes! Use command line:
   python cli.py --keywords "Developer" --locations "New York"
   See FEATURES.md for all CLI options.

Q: Is my data private?
A: Yes! Everything processes locally. Nothing uploaded to servers.
